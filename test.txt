Login 페이지 테스트 시나리오
1. 초기 렌더링
 이메일, 비밀번호 입력 필드와 로그인 버튼이 정상 렌더링된다.
 로그인 버튼은 초기에는 비활성화 상태이다.
2. 입력 유효성 검사
✅ 이메일 유효성
 미입력 시: "ID를 입력해 주세요." 메시지 표시
 @kakao.com 형식이 아닐 경우: "@kakao.com 이메일 주소만 가능합니다." 메시지 표시
 유효한 이메일 입력 시: 에러 메시지 사라짐
✅ 비밀번호 유효성
 미입력 시: "PW를 입력해주세요." 메시지 표시
 8자 미만 입력 시: "PW는 최소 8글자 이상이어야 합니다." 메시지 표시
 유효한 비밀번호 입력 시: 에러 메시지 사라짐
3. 로그인 버튼 활성화
 이메일과 비밀번호가 모두 유효한 경우 로그인 버튼이 활성화된다.
4. 로그인 시도
✅ 성공
 로그인 함수가 올바른 인자로 호출됨
 sessionStorage에 email, name 저장
 "로그인이 완료되었습니다!" 토스트 메시지 표시
 메인 페이지(/)로 리디렉션
❌ 실패
 "로그인에 실패했습니다." 토스트 메시지 표시
 로그인 버튼 재활성화
 입력 필드 초기화되지 않음 (입력값 유지)
5. 로그인 상태 유지
 이미 로그인된 사용자가 로그인 페이지 접근 시 / 페이지로 리디렉션됨
🏆 RankingTabs 컴포넌트 테스트 시나리오
1. 초기 렌더링
 "실시간 급상승 선물랭킹" 제목이 표시된다.
 타겟 탭: 전체 / 여성 / 남성 / 청소년이 모두 표시된다.
 원하는 탭: 많이 찜한 / 많이 받은 / 많이 찜하고 받은이 모두 표시된다.
 기본 선택값:
타겟 탭: FEMALE
원하는 탭: MANY_WISH
 테스트 상품 목록이 정상 렌더링된다 (예: 상품 1, 2, 3 등).
2. 타겟 탭 기능
✅ 탭 클릭 동작
 전체 클릭 시:
mockSetSelected가 'ALL'로 호출됨
선택된 타겟이 'ALL'로 변경됨
 남성 클릭 시:
mockSetSelected가 'MALE'로 호출됨
선택된 타겟이 'MALE'로 변경됨
 청소년 클릭 시:
mockSetSelected가 'TEEN'으로 호출됨
선택된 타겟이 'TEEN'으로 변경됨
🎨 아이콘 표시 확인
 여성 탭: 👩 아이콘
 남성 탭: 👨 아이콘
 청소년 탭: 🧑 아이콘
3. 원하는 탭 기능
✅ 탭 클릭 동작
 많이 받은 클릭 시:

mockSetSelectedWantedTab이 'MANY_RECEIVE'로 호출됨
선택된 탭이 'MANY_RECEIVE'로 변경됨
 많이 찜하고 받은 클릭 시:

mockSetSelectedWantedTab이 'MANY_WISH_RECEIVE'로 호출됨
선택된 탭이 'MANY_WISH_RECEIVE'로 변경됨
4. 탭 조합 테스트
 타겟 탭과 원하는 탭을 연속 클릭해도 정상 작동한다.
남성 → 'MALE'
많이 받은 → 'MANY_RECEIVE'
 타겟 탭 순차 클릭: ALL → FEMALE → MALE → TEEN
 원하는 탭 순차 클릭: MANY_WISH → MANY_RECEIVE → MANY_WISH_RECEIVE
5. 커스텀 초기값 및 빈 목록 처리
 초기값으로 initialSelected="MALE"과 initialWantedTab="MANY_RECEIVE" 설정 시 해당 값으로 초기화된다.
 products=[]인 경우에도 제목과 탭은 정상 표시되고, 상품 목록은 비워진 상태로 표시된다.
